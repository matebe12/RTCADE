# Netplay 수동 검증 체크리스트

서버 쪽 변경이 포함되어 있으므로 먼저 개발 서버를 다시 시작한다.

- 프론트 + 서버: npm run dev:all
- 서버 변경 직후 기존 프로세스를 재사용하면 /api/rooms 가 반영되지 않을 수 있다.

## 1. 최근 플레이 / 즐겨찾기

1. 메뉴에서 방 만들기로 들어간다.
2. 아무 게임 하나를 즐겨찾기로 표시한다.
3. 같은 게임으로 방을 만들고, 실제 플레이 시작 직전이 아니라 게임 시작까지 진행한다.
4. 홈으로 돌아와 다시 방 만들기에 들어간다.

기대 결과:

- 최근 플레이 섹션에 방금 플레이한 게임이 보인다.
- 즐겨찾기 섹션에 별 표시한 게임이 보인다.
- 최근 플레이에 이미 노출된 게임은 즐겨찾기나 전체 목록에서 중복 노출되지 않는다.
- 검색어를 입력하면 최근 플레이 / 즐겨찾기 섹션이 숨고 검색 결과만 보인다.

## 2. 세션 요약

1. 브라우저 탭 2개를 연다.
2. 탭 A에서 방을 만들고, 탭 B에서 참가한다.
3. 실제 게임 시작까지 기다린다.
4. 탭 A 또는 탭 B 중 한쪽에서 나가기를 누른다.

기대 결과:

- 양쪽 모두 즉시 메뉴로 튕기지 않고 세션 요약 화면으로 이동한다.
- 게임 이름, 상대 프로필, 플레이 시간, 종료 사유가 표시된다.
- 다시 한 판을 누르면 같은 게임으로 새 방 생성 흐름이 시작된다.
- 다른 게임 고르기를 누르면 게임 브라우즈 화면으로 이동한다.

## 3. 공개 방 브라우저

1. 탭 A에서 방 만들기로 들어간다.
2. 상단 토글에서 공개 방을 선택한다.
3. 아무 게임이나 선택해 방을 만든다.
4. 탭 B에서 공개 방 둘러보기로 들어간다.

기대 결과:

- 탭 B 목록에 탭 A의 방이 보인다.
- 게임 이름, 시스템, 방 코드, 호스트 정보가 함께 보인다.
- 참가 버튼을 누르면 기존 코드 입력 없이 바로 방에 들어간다.
- 탭 B가 참가한 뒤에는 그 방이 공개 목록에서 사라진다.
- 탭 B가 나가고 탭 A가 계속 대기 중이면 그 방이 다시 공개 목록에 나타난다.

## 4. 비공개 방 확인

1. 탭 A에서 다시 방 만들기로 들어간다.
2. 상단 토글에서 초대 코드 방을 선택한다.
3. 아무 게임이나 선택해 방을 만든다.
4. 탭 B에서 공개 방 둘러보기를 새로고침한다.

기대 결과:

- 비공개 방은 공개 목록에 나타나지 않는다.
- 방 코드를 직접 입력한 경우에만 참가할 수 있다.

## 5. 최근 상대 / 빠른 참가

1. 탭 A와 탭 B로 실제 한 판을 시작한 뒤, 어느 한쪽에서 게임을 종료한다.
2. 종료한 탭을 홈으로 돌려보낸다.
3. 홈 메뉴를 확인한다.
4. 공개 방이 하나 이상 열려 있는 상태를 만든 뒤 홈 메뉴를 다시 확인한다.

기대 결과:

- 홈 메뉴에 최근 같이한 상대 섹션이 보인다.
- 상대 프로필, 마지막으로 같이한 게임, 최근 플레이 시점이 보인다.
- 같은 상대와 여러 번 플레이했다면 플레이 횟수가 누적되어 보인다.
- 마지막 종료 주체가 내가 종료인지 상대가 종료인지 함께 보인다.
- 다시 만나기 버튼을 누르면 공개 방 / 초대 코드 방 선택 다이얼로그가 뜬다.
- 초대 코드 방을 고르면 ROM 선택 화면 없이 바로 같은 게임 비공개 대기 방 생성으로 이어진다.
- 공개 방을 고르면 ROM 선택 화면 없이 바로 같은 게임 공개 대기 방 생성으로 이어진다.
- 홈 메뉴에 빠른 참가 섹션이 보이면 공개 방 일부가 바로 노출된다.
- 빠른 참가의 참가 버튼을 누르면 공개 방 브라우저를 거치지 않고 바로 방 참가가 시작된다.

## 6. 대기실 / 준비 완료 / 관전자 입장

1. 탭 A에서 방을 만든다.
2. 탭 B는 참가자로, 탭 C와 탭 D는 관전자로 같은 방 코드에 들어간다.
3. 참가자와 관전자 각각 준비 완료 버튼을 누른다.
4. 방장은 모든 인원이 준비 완료되기 전과 후의 시작 버튼 상태를 확인한다.
5. 게임 시작 후 새 탭에서 같은 코드로 관전 입장을 다시 시도한다.

기대 결과:

- 방장은 대기실에서 입장 인원 목록과 역할, 준비 상태를 볼 수 있다.
- 게스트와 관전자는 준비 완료 / 준비 취소를 전환할 수 있다.
- 관전자는 최대 5명까지만 입장할 수 있다.
- 방장은 게스트가 없거나 누군가 준비 중이면 게임을 시작할 수 없다.
- 모든 참가자가 준비 완료되면 방장에게 게임 시작 버튼이 활성화된다.
- 게임 시작 후에는 같은 코드로 새 관전자 입장이 거부된다.
- 게임 시작 후 역할 변경 UI는 노출되지 않는다.

## 7. 관전 연결 / 첫 프레임

1. 서로 다른 PC 또는 서로 다른 네트워크에 있는 브라우저 3개 이상을 준비한다.
2. PC A에서 방을 만들고, PC B는 참가자, PC C는 관전자로 게임 시작 전에 입장한다.
3. 세 명 모두 준비 완료 후 방장이 게임을 시작한다.
4. 가능한 경우 same LAN, 다른 Wi-Fi, 모바일 핫스팟까지 각각 반복한다.

기대 결과:

- 관전자 화면이 `연결 대기중` 상태에만 머무르지 않고 실제 첫 프레임이 도착한다.
- 영상 수신 전에는 `호스트 화면 수신 대기 중…` 또는 `첫 화면을 불러오는 중…`이 표시된다.
- 영상 재생이 잠깐 끊겨도 즉시 세션 종료로 가지 않고 `화면 재생을 복구하는 중…`으로 전환된다.
- 관전 중 상단 연결 배지는 영상이 오기 전까지 연결 중으로 유지되고, 첫 프레임 이후 연결됨으로 바뀐다.
- same LAN 관전과 일반 guest 참가 흐름에 회귀가 없다.

## 8. 홈 인기 게임 총 플레이 시간

1. 혼자하기 또는 호스트로 한 판 이상 실제 게임 시작까지 진행한다.
2. 10초 이상 플레이한 뒤 정상 종료한다.
3. 홈 화면으로 돌아와 오늘 / 주간 / 월간 인기 게임 카드를 확인한다.
4. 가능하면 플레이 중인 세션이 있는 상태와 종료 직후 상태를 둘 다 확인한다.

기대 결과:

- 인기 게임 카드 전체에 `N회 플레이 · 총 플레이 시간` 형식이 표시된다.
- 막 끝낸 세션의 시간이 홈 카드에 반영된다.
- 플레이 횟수 기준 정렬이 유지되고, 총 플레이 시간은 보조 정보로 표시된다.
- spectator 관전만 한 경우에는 인기 게임 총 플레이 시간이 증가하지 않는다.
